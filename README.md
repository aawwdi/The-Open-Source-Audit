# The Open Source Audit - OSS Capstone Project

--> Overview
[cite_start]This repository contains my submission for **The Open Source Audit**, a capstone project for the Open Source Software (OSS) NGMC Course on VITyarthi[cite: 1, 4]. The project consists of two main components:
1. [cite_start]A structured audit and comprehensive report of a chosen open-source software project[cite: 12].
2. [cite_start]A collection of five shell scripts demonstrating practical Linux command-line skills and automation[cite: 14, 15].

**Chosen Software:** [VLC Media Player]

---

--> Project Report
[cite_start]The written report is 12 to 16 pages long and covers the following aspects of the chosen software[cite: 29]:
* [cite_start]**Origin and Philosophy (Part A):** Exploration of the problem the software solves, its origin story, a detailed analysis of its open-source license, and the ethical philosophy behind its creation[cite: 32, 41, 42, 52].
* [cite_start]**Linux Footprint (Part B):** Documentation of the software's installation process, key directories, user permissions, and service management on a Linux system[cite: 63, 67].
* [cite_start]**The FOSS Ecosystem (Part C):** Mapping the dependencies, community governance, and the project's place within the broader open-source ecosystem[cite: 73, 75, 78].
* [cite_start]**Open Source vs Proprietary (Part D):** A critical analysis comparing the chosen software against its closest proprietary alternative across cost, security, support, and freedom[cite: 80, 81, 82].

[cite_start]*(Note: The final PDF report is uploaded directly to the VITyarthi portal alongside this repository URL[cite: 223, 225].)*


--> Shell Scripts
[cite_start]This repository includes five documented `.sh` scripts built to run on a Linux system[cite: 90, 92]. 

* [cite_start]**`script1_system_identity.sh` (System Identity Report):** Introduces the Linux system by displaying the distribution name, kernel version, current user, uptime, and the OS's open-source license[cite: 93, 94, 95, 96, 97, 98].
* [cite_start]**`script2_package_inspector.sh` (FOSS Package Inspector):** Checks if the chosen software is installed, retrieves its version, and prints a philosophy note about its purpose based on the package name[cite: 125, 126].
* [cite_start]**`script3_disk_permission.sh` (Disk and Permission Auditor):** Loops through vital system directories to report their disk space usage, owners, and permissions[cite: 145, 146, 147].
* [cite_start]**`script4_log_analyzer.sh` (Log File Analyzer):** Reads a specified log file line by line to count and summarize the occurrences of a specific keyword (e.g., ERROR or WARNING)[cite: 163, 164].
* [cite_start]**`script5_manifesto_generator.sh` (The Open Source Manifesto Generator):** An interactive script that asks the user three questions about their view on open source and generates a personalized philosophy statement saved to a `.txt` file[cite: 185, 186].
